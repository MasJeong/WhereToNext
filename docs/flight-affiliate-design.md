# 한눈에 보기

- 이 문서는 `떠나볼래`의 1차 제휴 수익화를 `항공권 비교 링크`로 도입하기 위한 구현 기준입니다.
- 첫 파트너는 `Skyscanner`, 운영이 막히면 같은 위치와 로그 구조를 유지한 채 `Trip.com`으로 전환합니다.
- 1차 노출 위치는 `목적지 상세 페이지` 하나로 제한합니다.
- 추천 결과와 제휴 링크는 분리하고, `제휴 링크 포함` 고지를 CTA 근처에 항상 둡니다.

# 1차 목표

- 추천 직후 사용자가 가장 자연스럽게 이어갈 수 있는 다음 행동으로 `항공권 비교`를 제공합니다.
- 항공권 제휴 링크는 목적지별 deeplink로 연결합니다.
- 클릭 로그를 저장해 목적지별 CTR과 파트너 반응을 확인합니다.

# 구현 기준

- 새 제휴 데이터는 `src/lib/affiliate/catalog.ts`에서 관리합니다.
- 항공권 링크 계산은 `src/lib/affiliate/service.ts`에서 처리합니다.
- UI는 `FlightAffiliatePanel` 하나로만 노출합니다.
- 목적지 상세에서만 보여 주고, 추천 결과 리스트에는 아직 넣지 않습니다.
- 로깅 API는 `POST /api/affiliate/clicks`로 둡니다.

# 파트너 운영 원칙

- 기본 파트너: `Skyscanner`
- fallback 파트너: `Trip.com`
- 전환 조건 예시:
  - 제휴 승인 거절
  - 트래픽/정책 조건 불일치
  - deeplink 추적 안정성 문제
  - 2주 이상 온보딩 정체

# 정책 기준

- CTA 가까이에 `제휴 링크 포함`과 `외부 예약 페이지로 이동하며, 예약이나 구매가 발생하면 수수료를 받을 수 있어요.`를 함께 둡니다.
- 외부 링크는 `rel="sponsored noopener noreferrer"`를 사용합니다.
- 추천 이유와 제휴 패널은 같은 카드로 섞지 않습니다.
- 사용자 후기나 공개 여행 기록 본문에는 제휴 링크를 숨겨 넣지 않습니다.

# 1차 측정 지표

- 목적지 상세 조회 수
- 제휴 CTA 클릭 수
- 목적지별 CTR
- 파트너별 CTR
- 로그인 여부별 CTR
