# 인터랙티브 지도 전환 계획

## 목표
- 지도 드래그/줌이 가능한 여행 지원 경험을 제공한다.
- `Maps JavaScript API` 비용이 자동으로 누적되지 않도록 클릭 후 로드 구조로 바꾼다.
- 결과, 상세, 복원 화면이 같은 여행 보조 정보와 주변 장소 요약을 재사용하게 만든다.

## 범위
- 포함 범위:
  - `DestinationTravelSupplement` 지도 계약을 좌표 기반 구조로 변경
  - 여행 보조 정보 DB 캐시 테이블 추가
  - 결과/상세/복원 화면의 지도 UI 교체
  - 공개 지도 키와 서버 Places 키 분리 문서화
- 제외 범위:
  - e2e 실패 케이스 전체 수정
  - 지도 노출 분석 이벤트 추가
  - 주변 장소 외의 별도 맛집/메뉴 전용 외부 API 도입

## 실행 계획
1. 여행 보조 정보 계약과 서비스 계층을 캐시 가능 구조로 바꾼다.
2. 클릭 전에는 정적 프리뷰, 클릭 후에는 JS 지도를 여는 카드 컴포넌트를 만든다.
3. 결과/상세/복원 화면이 같은 supplement 데이터를 쓰도록 route-data를 정리한다.
4. 관련 unit, lint, build를 검증하고 남은 e2e 실패는 후속 작업으로 기록한다.

## 미완료 후속 작업
1. `tests/e2e/account-future-trips.spec.ts`의 저장 추천 -> 앞으로 갈 곳 전환 실패 원인을 다시 확인한다.
2. `tests/e2e/auth/social-login.spec.ts`의 저장 재개 실패 원인을 현재 저장 게이트 흐름 기준으로 정리한다.
3. `tests/e2e/ios-acquisition-flow.spec.ts`의 소셜 로그인 후 복원/비교 보드 실패를 최신 복원 경로 기준으로 맞춘다.
4. `tests/e2e/recommendation-flow.spec.ts`의 signed-out 저장 리다이렉트 실패를 현재 인증 게이트 동작에 맞게 갱신한다.
