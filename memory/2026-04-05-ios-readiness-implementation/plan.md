# 한눈에 보기

- iOS 배포 준비 문서에서 실제 구현이 필요한 항목을 코드로 반영했다.
- 이번 범위는 개인정보처리방침 접근 경로, 계정 삭제, shell export 확인이다.

# iOS 준비 선행 구현

## 목표
- App Store 심사 전 바로 확인 가능한 정책 항목을 앱 안에서 사용할 수 있게 만든다.
- iOS shell 준비 상태를 문서가 아니라 실제 빌드 결과로 확인한다.

## 범위
- 포함 범위:
  - `/privacy` 페이지 추가
  - 공용 shell과 계정 설정 탭의 정책 진입점 추가
  - 계정 삭제 API / UI / 저장소 정리 구현
  - `npm run shell:build` 검증 및 문서 상태 갱신
- 제외 범위:
  - Capacitor 설정
  - iOS 네이티브 프로젝트 생성
  - App Store Connect metadata 입력

## 계획
1. 정책/계정 관련 현재 구현 상태를 점검한다.
2. 개인정보처리방침 노출 경로와 계정 삭제를 구현한다.
3. 관련 unit, lint, build, shell build를 검증한다.
4. iOS 준비 문서와 작업 기록을 현재 상태로 갱신한다.

## 리스크
- 계정 삭제 범위는 현재 저장소 구조 기준이며, 이후 신규 사용자 데이터가 생기면 함께 확장해야 한다.
- App Privacy 입력표와 TestFlight 검증은 아직 별도 후속 작업이 필요하다.
