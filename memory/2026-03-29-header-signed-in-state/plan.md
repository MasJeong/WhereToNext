# 한눈에 보기

- 목표: 로그인 뒤 상단 헤더에서 현재 계정 상태가 더 분명하게 보이도록 바꾼다.
- 범위: 상단 헤더 인증 영역의 시각 구조, 로그인 상태 라벨, 관련 e2e 검증을 포함한다.
- 기준: Booking/Airbnb류의 프로필 cluster와 Material top app bar 계정 액션 패턴을 참고한다.

## 계획

1. 현재 헤더에서 로그인 상태가 왜 약하게 보이는지 확인한다.
2. 이름/이니셜/상태를 묶은 profile chip으로 상단 우측 영역을 재구성한다.
3. 기존 auth e2e에 상태 표시 검증을 추가한다.
