# 한눈에 보기

- 상단 헤더 인증 영역을 세션 기반으로 바꿨다.
- 로그인 전에는 `로그인`, 로그인 후에는 `로그아웃`이 보이도록 정리했다.

## 변경 내용

### 1. 헤더 인증 액션 분리

- `src/components/trip-compass/shell-auth-nav.tsx`를 추가했다.
- 클라이언트에서 `/api/auth/session`을 조회해 세션이 있으면 `로그아웃`, 없으면 `로그인`을 보여준다.

### 2. 헤더 문구 정리

- 로그인 사용자는 텍스트 링크를 `내 여행 기록`으로 보게 했다.
- 비로그인 사용자는 기존처럼 `여행 기록`과 `로그인`을 보게 했다.
- 세션 확인 중에는 잘못된 CTA를 잠깐 보여주지 않도록 `확인 중` 상태를 둔다.

### 3. 로그인 후 헤더 검증 보강

- 소셜 로그인 e2e에서 계정 화면 진입 후 헤더 CTA가 `로그아웃`으로 바뀌는지 확인을 추가했다.
