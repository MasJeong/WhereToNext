# 한눈에 보기

- 목표: 여행 기록 화면을 기록 조회 중심으로 다시 짜고, `기록 추가`는 별도 step 화면으로 분리한다.
- 범위: 여행 기록 리스트 UI, 별도 추가 화면, 이미지 업로드, 메모 저장, 저장한 추천 탭 분리, 관련 데이터 계약과 검증까지 포함한다.
- 제외: 추천 엔진 로직과 소셜 로그인 공급자 연동 자체는 이번 범위에서 건드리지 않는다.

## 계획

1. 기존 계정 화면과 여행 기록 데이터 모델을 확인한다.
2. 여행 기록을 리스트 중심으로 다시 구성하고 저장한 추천을 별도 탭으로 분리한다.
3. `/account/history/new` step 화면을 만들고 이미지 업로드, 메모 저장을 붙인다.
4. 데이터 계약, DB 스키마, API, e2e 검증까지 맞춘다.
