# 한눈에 보기

- 목표: 홈 질문 퍼널 변경 이후 실패하던 e2e 회귀를 현재 구현 기준으로 다시 맞춘다.
- 범위: `auth/social-login`, `ios-acquisition-flow`, `recommendation-flow`의 홈 진입 헬퍼와 실패 케이스.
- 완료 기준: 축소 Playwright 재현이 통과하고, 전체 `test:e2e`가 다시 녹색으로 돌아온다.

## 작업 계획

1. 현재 홈 질문 퍼널의 실제 단계 수와 `다음` 버튼 개입 지점을 코드로 다시 확인한다.
2. 깨지는 e2e 파일들의 헬퍼와 개별 실패 케이스를 현재 퍼널 순서에 맞게 수정한다.
3. 축소 Playwright 실행으로 회귀가 정리됐는지 먼저 확인한다.
4. 전체 `test:e2e`와 `test:unit`으로 최종 검증한다.
