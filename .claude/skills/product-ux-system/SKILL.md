# 🎯 목적

이 스킬의 목적은 단순한 UI 개선이 아니라  
**실제 서비스 수준의 UX 설계**를 강제하는 것이다.

---

# ❗ 금지 사항

다음 패턴은 절대 사용하지 마라:

- 모든 영역을 카드로 감싸는 구조
- padding + border 남용
- Dribbble 스타일 디자인
- “예쁘게 만드는 것”만 목표로 하는 UI

---

# ✅ 필수 설계 순서 (반드시 지켜라)

UI를 만들기 전에 반드시 아래 순서를 따른다:

1. 사용자 목표 정의
2. 사용자 행동 흐름 설계
3. 화면의 단일 목적 정의
4. 정보 위계 구조 설계
5. UI 구성

👉 UI는 마지막 단계다.

---

# 🧠 UX 핵심 원칙

1. 한 화면 = 한 목적
2. CTA는 항상 1개만 강조
3. 선택지는 최대 3개 이하
4. 텍스트가 아니라 구조로 설명
5. 감정 → 행동 흐름을 만든다

# 📊 벤치마크 기준

다음 서비스 수준을 기준으로 설계한다:

- 토스 → 정보 흐름 중심
- 당근 → 리스트 기반 UX
- Airbnb → 카드 최소화
- 트리플 → 감정 + 추천 흐름

👉 이 수준보다 낮으면 실패다.

---

# ⚙️ 출력 요구사항

반드시 포함:

1. Before / After 구조 비교
2. 사용자 흐름 설명
3. 화면별 UX 설계 이유
4. React 컴포넌트 코드

---

# 🔥 핵심 요약

UI는 디자인이 아니라 결과다.  
UX 설계를 먼저 하지 않으면 결과는 무조건 망한다.
