# 🎯 목적

이 스킬의 목적은 단순한 UI 개선이 아니라  
**SooGo를 실제 여행 추천 서비스처럼 보이고 작동하게 만드는 UX 설계**를 강제하는 것이다.

이 스킬은 다음을 기본값으로 둔다.

- 시각보다 **정보 위계**를 먼저 설계한다.
- 분위기보다 **신뢰감과 결정 지원**을 우선한다.
- 한국어 사용자-facing 문구는 한국 여행자 기준으로 쓴다.
- Instagram/소셜 분위기 레이어는 보조로만 쓰고 추천 근거를 흐리지 않는다.
- 모바일에서 빠르게 이해하고 바로 다음 행동으로 갈 수 있어야 한다.

---

# ❗ 금지 사항

다음 패턴은 절대 사용하지 마라:

- 모든 영역을 카드로 감싸는 구조
- padding + border 남용
- Dribbble 스타일 디자인
- “예쁘게 만드는 것”만 목표로 하는 UI
- 정보보다 배경/효과가 먼저 보이는 hero
- CTA를 여러 개 동시에 같은 강도로 강조하는 구조
- 헤더가 화면 목적보다 더 튀는 구조
- 영어 감성 카피를 한국어 핵심 메시지보다 크게 두는 것
- Instagram 근거를 추천 엔진처럼 보이게 하는 것
- 로고/브랜드를 과장해 제품보다 브랜딩 쇼케이스처럼 만드는 것

# ✅ 필수 설계 순서 (반드시 지켜라)

UI를 만들기 전에 반드시 아래 순서를 따른다:

1. 사용자 목표 정의
2. 사용자 행동 흐름 설계
3. 현재 화면의 단일 목적 정의
4. 사용자의 다음 행동 정의
5. 핵심 정보 3개 이하로 정리
6. 정보 위계 구조 설계
7. CTA 우선순위 고정
8. UI 구성

👉 UI는 마지막 단계다.

---

# 🧠 UX 핵심 원칙

1. 한 화면 = 한 목적
2. CTA는 항상 1개만 강조
3. 선택지는 최대 3개 이하
4. 텍스트가 아니라 구조로 설명
5. 감정 → 행동 흐름을 만든다
6. 설렘보다 먼저 이해되고, 이해 다음에 납득되어야 한다
7. 분위기 레이어는 추천 이유를 덮으면 안 된다

---

# ✈️ SooGo 제품 UX 해석

SooGo는 일반 여행 콘텐츠 사이트가 아니다.
핵심은 **여행지 탐색 피로를 줄이고, 빠르게 다음 행동으로 보내는 추천 제품**이다.

따라서 모든 화면은 아래를 만족해야 한다.

1. 지금 무엇을 도와주는지 바로 이해된다.
2. 왜 이 추천/선택이 맞는지 짧게 납득된다.
3. 다음 행동이 하나로 보인다.
4. 저장, 비교, 복귀 같은 보조 행동은 메인 행동을 방해하지 않는다.

이 흐름을 깨는 디자인은 실패다.

# 🔠 타이포그래피 규칙

타이포그래피는 감성이 아니라 **정보 통제 수단**이다.

반드시 지킬 것:

- type scale은 적게 쓴다.
- heading / section title / body / helper / caption 역할을 명확히 나눈다.
- 스타일이 아니라 역할 이름으로 관리한다.
- body text는 모바일에서도 읽기 쉬워야 한다.
- line-height는 읽기 편한 범위로 유지한다.

역할별 기준:

- **Hero / Screen Title**: 무엇을 도와주는지가 먼저 보인다.
- **Section Title**: 명사형 우선, 빠르게 훑을 수 있어야 한다. 예: `추천 이유`, `체크할 점`, `여행 정보`
- **Body**: 짧고 단단하게, 1문장 1생각 원칙
- **Helper / Caption**: 컨트롤 이름 반복 금지, 왜 중요한지 설명

금지:

- light weight를 중요한 텍스트에 쓰는 것
- 연한 회색으로 body text를 처리하는 것
- heading 수가 너무 많아 hierarchy가 무너지는 것
- 영문 감성 문구를 한국어 핵심 메시지보다 크게 쓰는 것

# 📝 한국어 카피 규칙

`docs/korean-copy-guidelines.md`를 기본 규칙으로 따른다.

반드시 지킬 것:

- 한국어 우선
- 짧고 명확하게
- 따뜻하되 과하게 귀엽지 않게
- 자신감 있고 실용적으로
- Recommendation-first
- Anonymous-first 문맥 유지

UI 문구 규칙:

- 버튼은 행동이 먼저 보여야 한다. 예: `추천 받기`, `비교하기`, `저장하기`
- 섹션 제목은 명사형 우선
- 에러/빈 상태는 무엇이 안 됐는지 말하고 다음 행동을 하나만 제안한다.

하지 말 것:

- 영어 원문 직역
- 느낌표, 유행어, 과장형 마케팅 톤 남발
- 제품이 제공하지 않는 것을 약속하는 문구
- Instagram 근거를 ranking authority처럼 보이게 하는 표현

---

# 🧭 헤더 / 내비게이션 규칙

헤더는 브랜딩 쇼케이스가 아니라 **위치 확인 + 빠른 행동 + 계정 접근**을 담당한다.

원칙:

- 브랜드, primary CTA, account/auth의 3축만 유지한다.
- 메뉴 수를 늘리기보다 핵심 행동을 더 선명하게 한다.
- active / hover / focus state는 모두 명확해야 한다.
- 모바일에서 brand row와 action row를 분리하는 것은 허용되지만, 숨김 메뉴 남발은 지양한다.

금지:

- 로고가 CTA보다 더 튀는 것
- 모든 메뉴를 동등한 시각 강도로 두는 것
- hover에만 의존하는 affordance
- 데스크톱 한 줄 구성만 고집하다 모바일 가독성을 깨는 것

---

# 🎨 브랜드 / 감정 UX 규칙

브랜드는 장식이 아니라 신뢰 신호다.

반드시 지킬 것:

- 로고는 식별이 목적이지 장식이 목적이 아니다.
- clear space, minimum size, contrast를 지킨다.
- 브랜드 표현은 제품 행동을 가리면 안 된다.
- 여행 감성은 로고 과장이 아니라 색, spacing, 타이포, 이미지 밀도로 만든다.

SooGo 감정 흐름:

1. 설렘
2. 기대
3. 안심
4. 결정

즉, 화면은 예쁜 것만으로 끝나면 안 되고,
**설렘 → 기대 → 납득 → 행동**으로 이어져야 한다.

금지:

- 감성 카피만 있고 행동 설계가 없는 화면
- 체크할 점보다 분위기 묘사가 먼저 보이는 결과 카드
- 소셜 이미지를 추천 근거처럼 과장하는 구조

# 🔥 핵심 요약

UI는 디자인이 아니라 결과다.  
SooGo의 좋은 UI는 화려한 UI가 아니라,
**빠르게 이해되고, 믿을 수 있고, 바로 다음 행동으로 가게 만드는 UI**다.

UX 설계를 먼저 하지 않으면 결과는 무조건 망한다.
