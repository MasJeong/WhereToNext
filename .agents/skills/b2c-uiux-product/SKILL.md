# B2C UI/UX Product

## 개요

이 스킬은 소비자 대상 제품 화면에서 전환, 안내, 신뢰, 재방문을 함께 개선해야 할 때 사용한다. 막연한 미감보다 구체적인 제품 의사결정을 우선한다.

## 사용 시점

- 사용자가 B2C 제품의 UI, UX, 온보딩, 랜딩 페이지, 결제, 예약, 검색, 추천, 리텐션, 모바일 사용성 개선을 요청할 때
- 과제가 소비자용 화면, 퍼널, 기능과 관련되어 있고 명확성, 신뢰, 속도, 전환이 중요한 경우
- 정보 구조, 마찰, 빈 상태, CTA, 제품 카피 구조에 대한 피드백이나 개선이 필요한 경우

다음에는 이 스킬을 쓰지 않는다.

- 제품 흐름과 무관한 순수 브랜딩 비주얼 작업
- B2C 사용자 흐름과 직접 연결되지 않는 백오피스형 엔터프라이즈 대시보드 작업
- 사용자 여정에 영향이 없는 백엔드 아키텍처나 API 설계 작업

## 작업 원칙

1. 컴포넌트 미감보다 사용자 여정에서 출발한다.
2. 화면의 주 행동을 먼저 식별하고, 구조와 시각 위계 모두에서 가장 강하게 드러낸다.
3. 역할이 없는 단계, 선택지, 카피는 제거한다.
4. 과장보다 신뢰를 우선한다. 근거, 미리보기, 기본값, 가격 명확성, 안전성, 되돌릴 수 있음이 중요하다.
5. 소비자용 화면은 기본적으로 모바일 우선으로 판단한다.
6. 기존 디자인 시스템이 있으면 존중하고, 없으면 평범한 기본형 대신 의도적인 방향을 잡는다.
7. 변경 제안에는 사용자 문제, 제품 관점의 이유, 기대 행동 변화를 함께 설명한다.

## 작업 흐름

### 1. 현재 화면 이해

먼저 아래를 정리한다.

- 사용자가 누구인지
- 지금 이 화면에서 무엇을 하려는지
- 비즈니스가 다음으로 유도하고 싶은 행동이 무엇인지
- 사용자가 망설이거나 이탈하거나 불신하게 되는 지점이 무엇인지

### 2. 현재 UX 진단

현재 UI에서 아래를 점검한다.

- 위계가 약한지
- 주의가 너무 많은 행동으로 분산되는지
- 히어로 영역이 정보 밀도가 낮은지
- 신뢰 신호가 숨어 있는지
- 라벨이나 카테고리 이름이 헷갈리는지
- 가치를 이해하기까지 스크롤이 너무 긴지
- 입력, 가입, 예약, 결제, 복원 흐름에 마찰이 큰지

평가 기준은 `references/heuristics.md`를 읽고 적용한다.

### 3. 우선 목표 선택

다음 중 현재 화면의 1차 목표를 먼저 정한다.

- 전환
- 활성화
- 리텐션
- 신뢰 형성
- 가치 도달 속도

목표가 충돌하면, 현재 화면의 역할에 가장 가까운 목표를 우선한다.

### 4. 변경 수행

아래를 직접 개선하는 변경을 우선한다.

- 다음 행동의 명확성
- 왜 이 제품을 믿어도 되는지에 대한 근거
- 사용자가 결심하기 전에 결과를 볼 수 있는 정도
- 체감 노력
- 오류나 불확실성에서 회복하는 능력

### 5. 검증

결과를 아래 기준으로 확인한다.

- 핵심 CTA가 몇 초 안에 보이는지
- 모든 문장을 읽지 않아도 핵심 가치가 드러나는지
- 모바일에서도 상태 변화가 이해되는지
- 카피가 스캔 가능하고 구체적인지
- 흐름이 불확실성을 줄이는지, 오히려 늘리는지

랜딩, 온보딩, 가격, 결제, 검색, 추천, 계정 상태 같은 공통 플로우는 `references/flows.md`를 읽고 맞춘다.

## 출력 방식

리뷰하거나 제안할 때는:

- 가장 중요한 제품 리스크나 UX 실패부터 말한다.
- 각 제안을 사용자 행동이나 비즈니스 결과와 연결한다.
- 이유 없는 "더 깔끔하게", "여백을 개선하자" 같은 모호한 말은 피한다.
- "CTA 경로를 둘에서 하나로 합친다", "결제 전 환불 정책을 먼저 노출한다", "추천 옵션에 근거 있는 기본값을 준다"처럼 구체적으로 말한다.

UI를 직접 구현할 때는:

- 경쟁하는 행동 수를 낮춘다.
- 타이포, 여백, 대비, 밀도로 우선순위를 빠르게 만든다.
- 로딩, 빈 상태, 에러, 성공 상태를 모두 의도적으로 설계한다.
- 저장소 패턴상 필요하면 테스트 가능한 selector도 함께 추가한다.

## 참고 자료

- `references/heuristics.md`에는 평가 기준과 판단 원칙이 있다.
- `references/flows.md`에는 랜딩, 온보딩, 가격, 결제, 검색, 추천, 계정 상태 체크리스트가 있다.
