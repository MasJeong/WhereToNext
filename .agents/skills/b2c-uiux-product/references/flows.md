# 공통 B2C 플로우

현재 작업과 맞는 섹션만 읽고 적용한다.

## 랜딩 페이지

- 가치 제안을 첫 화면에서 바로 말한다.
- 핵심 CTA 하나와 보조 경로 하나 정도만 둔다.
- 수치, 예시, 미디어, 후기, 익숙한 출처 같은 근거를 빠르게 붙인다.
- 모호한 히어로 카피를 줄이고 구체적인 결과 중심 문장으로 바꾼다.

## 온보딩

- 다음 경험을 바꾸는 정보만 묻는다.
- 긴 초기 입력 대신 점진적 공개를 쓴다.
- 답변을 제품이 어떻게 활용하는지 보여준다.
- 막다른 확인 화면이 아니라 즉시 보상을 주는 결과로 끝낸다.

## 가격 또는 예약

- 총비용을 초반부터 읽기 쉽게 보여준다.
- 제한 조건, 환불 규칙, 중요한 조건을 결심 전에 노출한다.
- 예상 밖 수수료와 설명 없는 옵션을 줄인다.
- 추천 플랜이나 패키지는 이유와 함께 보여준다.

## 결제

- 결제 결정 주변의 시각적 노이즈를 줄인다.
- 신뢰 신호를 CTA 가까이에 둔다.
- 에러 상태는 구체적이고 복구 가능하게 만든다.
- 사용자가 무엇을 구매하는지 맥락이 유지되게 한다.

## 검색 또는 탐색

- 필터는 이해하기 쉽고 되돌리기 쉽게 만든다.
- 유용한 기본값을 보여준다.
- 사용자를 탓하는 빈 상태를 피한다.
- 사용자가 압도되지 않으면서 선택지를 좁힐 수 있게 돕는다.

## 추천 화면

- 왜 1순위 결과가 맞는지 설명한다.
- 신뢰할 수 있을 정도의 근거를 붙인다.
- 대안은 보이게 두되 보조 위치로 둔다.
- 확신이 낮으면 솔직하게 말하고 우회 경로를 준다.

## 저장 상태 또는 계정 화면

- 복원과 이어하기 경로를 분명하게 만든다.
- 무엇이 언제 저장됐는지 보여준다.
- 조용한 데이터 손실을 막는다.
- 복원이 불가능하면 분명하게 실패를 알리고 다음 최선 행동을 제시한다.
